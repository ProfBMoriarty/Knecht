API | Config

These commands set constants specific to the application so that they do not have to be passed with every request.

  • setAddress ( address )
  • setApplication ( app )
 

setAddress( address )

setAddress() specifies the URL or IP address of the server to message and establishes a connection.

Parameters:

  1. address : string

The required address parameter must be the address of the server.

Usage notes

1. setAddress() is required for any function that accesses the server. If it is not called first, those functions will fail.

2. Note that this function does not verify the address given, so if it is invalid an error will not appear until another function tries to access it.

Return value

setAddress() returns nothing.

 

setApplication( app )

setApplication() specifies the name of the application in the name of which queries will be made.

Parameters:

  1. app : string

The required app parameter must be unique to the application

Usage notes

1. setApplication() is required for any function that handles data on the server. If it is not called first, those functions will fail.

Return value

setAddress() returns nothing.

API | User Accounts

These commands manage user accounts.

  • checkEmail ( email, callback )
  • register ( email, password, callback )
  • login ( email, password, callback )
  • recoverPassword ( email, callback )
  • changePassword ( password, callback )
  • unregister ( callback )
  • logout ( callback )
 

checkEmail( email, callback )

checkEmail() checks whether the given email is already the name of an existing account.

Parameters:

  1. email : string
  2. callback : function

The required email parameter should correspond to an existing email address.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member result that is a response code.

result is K.OK if the given email is not yet in use.

result is K.INVALID if the given email is in use.

result is K.ERROR the server could not be accessed or the response was corrupted.

Return value

checkEmail() returns nothing.

 

register( email, password, callback )

register() establishes a new account on the server.

Parameters:

  1. email : string
  2. password : string
  3. callback : function

Returns: nothing

The required email parameter must be valid email address unique from all other email address in the system.

The required password parameter is a string that will be used to authorize users to use this account in the future.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member result that is a response code.

result is K.OK if the account was successfully created.

result is K.INVALID if the account could not be created because the given email was in use.

result is K.ERROR the server could not be accessed or the response was corrupted.

Return value

register() returns nothing, but the callback function is called asynchronously when it completes.

 

login ( email, password, callback )

login() creates a session on the server and stores a key (handled by the client) so that account specifically requests can be made.

Parameters:

  1. email : string
  2. password : string
  3. callback : function

Returns: nothing

The email parameter indicate the username given to the server, and the password parameter indicate the password given to the server. Both strings are required.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member result that is a response code.

result is K.OK if the client is now logged in to the specified account.

result is K.UNAUTH if the given password is not correct.

result is K.INVALID if the given email is not an existing account.

result is K.ERROR the server could not be accessed or the response was corrupted.

login() returns nothing. Upon completion of the ajax request, callback is called

 

recoverPassword( email, callback )

recoverPassword() sends an email with the account password to the email address specified at account creation.

Parameters:

  1. email : string
  2. callback : function

Returns: nothing

The required email parameter must correspond to an existing account.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member result that is a response code.

result is K.OK if the email was successfully sent.

result is K.INVALID if the account does not exist.

result is K.ERROR the server could not be accessed or the response was corrupted.

Return value

recoverPassword() returns nothing, but the callback function is called asynchronously when it completes.

 

changePassword( password, callback )

changePassword() change the password of the account currently logged into.

Parameters:

  1. password : string
  2. callback : function

Returns: nothing

The required password parameter will be used to access the account in the future.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member result that is a response code.

result is K.OK if the email was successfully sent.

result is K.UNAUTH if the account could not be authorized.

result is K.INVALID if the account does not exist.

result is K.ERROR the server could not be accessed or the response was corrupted.

Return value

changePassword() returns nothing, but the callback function is called asynchronously when it completes.

 

unregister( callback )

unregister() deletes the current account from the server and all associated data.

Parameters:

  1. callback : function

Returns: nothing

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member result that is a response code.

result is K.OK if the account was successfully deleted.

result is K.UNAUTH if the account could not be authorized.

result is K.INVALID if the account does not exist.

result is K.ERROR the server could not be accessed or the response was corrupted.

Usage notes

1. This action can not be undone. Do not call it lightly.

Return value

unregister returns nothing, but the callback function is called asynchronously when it completes.

 

logout ( image )

logout() expires the session.

Parameters:

  1. callback : function

The callback parameter is called with a string that is the server's response upon completion.

Return value

logout() returns nothing

API | User Data

These commands control the creation, management and retrieval of data as used by a single application.

  • putData ( field, data, callback )
  • getData ( field, callback )
  • deleteData ( field, callback )
 

putData( field, data, callback )

putData() saves arbitrary data to the server.

Parameters:

  1. field : string
  2. data : string
  3. callback : function

Returns: nothing

The required field can be anything that does not already identify a body of data for this user and game, but the game will have to reproduce it to retrieve the data.

The required data can be anything that the game can itself parse.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member result that is a response code.

result is K.OK if the data was successfully posted.

result is K.UNAUTH if the account could not be authorized.

result is K.INVALID if the account does not exist.

result is K.ERROR the server could not be accessed or the response was corrupted.

Return value

postData returns nothing, but the callback function is called asynchronously when it completes.

 

getData( field, callback )

getData() retrieves data to the server.

Parameters:

  1. field : string
  2. callback : function

Returns: nothing

The required field is a unique string used to identify the data.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member result that is a response code.

response is K.OK if the data was successfully retrieved.

response is K.UNAUTH if the account could not be authorized.

response is K.INVALID if the account does not exist.

response is K.ERROR the server could not be accessed or the response was corrupted.

The object passed to callback has a member body that is the retrieved data if it could be retrieved.

Return value

getData returns nothing, but the callback function is called asynchronously when it completes.

 

deleteData( field, callback )

deleteData() deletes the specified data from the server.

Parameters:

  1. field : string
  2. callback : function

Returns: nothing

The required field is the same name used as when the data is posted.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member response that is a response code.

response is K.OK if the data was successfully deleted.

response is K.UNAUTH if the account could not be authorized.

response is K.INVALID if the account does not exist.

response is K.ERROR the server could not be accessed or the response was corrupted.

Usage notes

1. If data with that key doesn't exist, nothing happens.

Return value

deleteData returns nothing, but the callback function is called asynchronously when it completes.

API | Group Host

These commands control the creation, management and retrieval of groups for managing data that can be access by multiple clients simultaneously. They can only be called by the group host.

  • startGroup ( group, grouppass, callback )
  • closeGroup ( group, callback )
  • addMember ( group, email, callback )
  • listMembers ( group, callback )
  • removeMember ( group, email, callback )
  • submitUpdate ( group field, data, callback )
  • getGroupData ( group, field, callback )
  • grantPermission ( group, user, field, callback )
  • revokePermission ( group, user, field, callback )
  • listenInputs ( group, callback )
 

startGroup( group, grouppass, callback )

startGroup() creates a new group with the calling account as the host.

Parameters:

  1. group : string
  2. grouppass : string
  3. callback : function

Returns: nothing

The required group is the same name that will be used to identify the new group.

The required grouppass is the group password. It is never referenced again.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member response that is a response code.

response is K.OK if the group was successfully created.

response is K.UNAUTH if the account could not be authorized.

response is K.INVALID if the account does not exist or a group with the given name already exists.

response is K.ERROR the server could not be accessed or the response was corrupted.

Usage notes

1. The new host must still call listenInputs to receive inputs from both accepted and perspective members of the group.

Return value

startGroup returns nothing, but the callback function is called asynchronously when it completes.

 

closeGroup( group, callback )

closeGroup() deletes an existing group and all data associated with it.

Parameters:

  1. group : string
  2. callback : function

Returns: nothing

The required group is the name of the group to be deleted.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member response that is a response code.

response is K.OK if the group was successfully deleted.

response is K.UNAUTH if the account could not be authorized.

response is K.INVALID if the account does not exist or a group with the given name does not exist.

response is K.ERROR the server could not be accessed or the response was corrupted.

Usage notes

1. This function cannot be undone.

Return value

deleteGroup returns nothing, but the callback function is called asynchronously when it completes.

The required email is the name of the account to be made a member of the group.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member response that is a response code.

response is K.OK if the member was successfully added.

response is K.UNAUTH if the account could not be authorized.

response is K.INVALID if the account does not exist or a group with the given name does not exist.

Returns: nothing

The required group is the name of the group the members of which are to be retrieved.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member response that is a response code.

response is K.OK if the list was successfully retrieved.

response is K.UNAUTH if the account could not be authorized.

The object passed to callback has a member body that is an array of strings that are the emails of the members of the group if they could be retrieved.

Return value

submitUpdate() places an object on the server and alerts all members so they can access it if necessary.

Returns: nothing

The required group is the name of the group the update is to be sent to.

The required field is the name of the data to be posted on the server in the group's name.

The required data is an object in a format that the members will be able to interpret.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member response that is a response code.

response is K.OK if the update was successfully posted.

response is K.UNAUTH if the account could not be authorized.

response is K.INVALID if the account does not exist or a group with the given name does not exist.

response is K.ERROR the server could not be accessed or the response was corrupted.

Return value

submitUpdate returns nothing, but the callback function is called asynchronously when it completes.

 

getGroupData( group, field, callback )

getGroupData() retrieves an object on the server previously posted by the submitUpdate function.

Parameters:

  1. group : string
  2. field : string
  3. callback : function

Returns: nothing

The required group is the name of the group the data is to be retrieved from.

The required field is the name of the data used when it was posted.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member response that is a response code.

response is K.OK if the data was successfully retrieved.

response is K.UNAUTH if the account could not be authorized.

response is K.INVALID if the account does not exist or a group with the given name does not exist.

response is K.ERROR the server could not be accessed or the response was corrupted.

The object passed to callback has a member body that is the requested data if it could be retrieved.

Usage notes

1. Any member of the group, not only the host, can call this function.

Return value

getGroupData returns nothing, but the callback function is called asynchronously when it completes.

 

listenInputs( group, callback )

listenInputs() returns a list of inputs from members both accepted and perspective.

Parameters:

  1. group : string
  2. callback : function

Returns: nothing

The required group is the name of the group to which inputs are to be listened for.

The required callback parameter must take an object.

The object passed to callback has a member timestamp that is a number representing the time the response was sent in unix time.

The object passed to callback has a member response that is a response code.

response is K.OK if the data was successfully retrieved.

response is K.UNAUTH if the account could not be authorized.

response is K.INVALID if the account does not exist or a group with the given name does not exist.

response is K.ERROR the server could not be accessed or the response was corrupted.

The object passed to callback has a member body that is an array of objects corresponding to pending inputs.

The objects of body each have a memberuser that is a string that is the email of the account that submitted the input.

The objects of body each have a memberinput that is an object representing the member's input.

The objects of body each have a membertime that is a number representing the time the response was sent in unix time.

Usage notes

1. The callback maybe called multiple times as new input is received because this function refreshes itself.

Return value

getGroupData returns nothing, but the callback function is called asynchronously when it completes.
